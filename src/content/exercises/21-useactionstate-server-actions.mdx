## Goal
Read a complete Server Action example that validates input, returns either a
success or an error, and wires up to the form via `useActionState`.

## Concepts
Server Actions erase the RPC boilerplate. You write the function, mark it
`'use server'`, and pass it to a form's `action` prop. `useActionState` wraps
the action on the client to give you the previous/current state and pending
status — ideal for displaying inline validation errors.

## Read-along
1. `actions.ts` — the server function.
2. `NewTodoForm.tsx` — client component using `useActionState`.

## Common pitfalls
- Returning non-serializable values — Server Action results must be serializable.
- Forgetting to give each form field a `name` attribute — `formData.get(name)` needs it.

## Stretch
Add a `pending` spinner by destructuring `[state, action, pending]` from
`useActionState`.
