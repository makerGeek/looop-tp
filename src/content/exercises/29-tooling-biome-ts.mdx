## Goal
Read annotated config files showing a Biome + strict TS + CI setup.

## Concepts
Fast feedback loops matter. Biome lints + formats in one pass, TS strict mode
catches bugs in dev, and a minimal CI runs all of that in under a minute.

## Read-along
1. `biome.json` — linter + formatter config.
2. `tsconfig.json` — strict mode + path aliases.
3. `.github/workflows/ci.yml` — typecheck, lint, test, build.

## Common pitfalls
- Leaving `skipLibCheck: false` and fighting third-party types forever.
- Running formatter and linter in separate steps — Biome does both.

## Stretch
Add a pre-commit hook via `simple-git-hooks` that runs `biome check --apply`.
