## Goal
Validate an email + password form using a Zod schema and infer the TS type.

## Concepts
Zod lets you declare the shape and constraints once, then derive both the
TypeScript type (`z.infer`) and a runtime parser. In production, pair it with
React Hook Form's `zodResolver` for zero boilerplate validation.

## Steps
1. Declare the schema.
2. On submit, run `schema.safeParse(values)` and render field errors.
3. Block submit until the form parses successfully.

## Common pitfalls
- Using `parse` (throws) instead of `safeParse` (returns a result).
- Re-declaring types manually instead of `z.infer`.

## Stretch
Add a `confirmPassword` field with a `.refine()` rule matching `password`.
