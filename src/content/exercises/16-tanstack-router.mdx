## Goal
Build a mini typed router: two "pages" and validated search params.

## Concepts
TanStack Router treats the URL as typed state. You declare each route's
`search` shape and get autocomplete + compile errors when constructing links.

## Steps
1. Represent `route = 'home' | 'search'` in state.
2. For `'search'`, require `{ q: string; page: number }`.
3. Validate inputs when navigating — reject invalid shapes.

## Common pitfalls
- Parsing `URLSearchParams` into `any` and losing type safety.

## Stretch
Add a `'detail'` route with `{ id: string }` and a typed link helper.
