## Goal
Swap between light and dark themes by toggling a class on the root, powered by
CSS custom properties.

## Concepts
Good theming has two layers: **tokens** (the palette, scale, radii) and
**semantic aliases** (`--color-primary`, `--color-bg`). Components reference the
aliases; only the aliases change between themes.

## Steps
1. Define a palette + semantic tokens in the component (to stand in for
   `@theme` — the sandbox doesn't ship with Tailwind).
2. Drive colors off `var(--color-*)`.
3. Toggle `document.documentElement.classList.toggle('dark')`.

## Common pitfalls
- Hard-coding hex values in components — makes theming impossible later.

## Stretch
Add a third theme (sepia) and show that the same components work unchanged.
