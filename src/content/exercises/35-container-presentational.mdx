## Goal
Split a "users dashboard" into a `UsersContainer` (fetches + state) and a
`UsersList` (pure render).

## Concepts
The container/presentational split is older than hooks but still useful when a
component owns side effects. Tests of the presentational half can run with
zero mocks, just static props.

## Steps
1. `types.ts` exports `User`.
2. `UsersList.tsx` accepts `{ users, onSelect }` props and renders the list.
3. `UsersContainer.tsx` simulates a fetch, stores selection state, and renders
   `<UsersList>` with handlers wired up.

## Common pitfalls
- Letting the presentational component grow side effects ("just one fetch") —
  the boundary breaks fast.

## Stretch
Add a `<UsersList />` story in `users-list.story.tsx` showing the component
with mocked data, with no container at all.
