## Goal
Render a list of todo objects and let the user filter by a text query.

## Concepts
Keys help React identify which items changed, moved, or were removed.
Derived state (the filtered array) should be computed during render, not stored
in `useState` — otherwise you risk it getting out of sync.

## Steps
1. Seed state with a few todo items: `{ id, label, done }`.
2. Add a search `<input>` and compute a filtered view by `query`.
3. Render the filtered list as `<ul>` with one `<li key={item.id}>` per item.

## Common pitfalls
- Using the array index as a key when items can be reordered or removed.
- Storing `filtered` in state and forgetting to update it.

## Stretch
Add a "hide completed" checkbox that further filters the list.
