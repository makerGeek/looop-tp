## Goal
Build a small "shopping cart" state machine split across files: `types.ts`,
`actions.ts`, `reducer.ts`, and a `CartView.tsx` consumer.

## Concepts
At scale, reducers become unreadable when state, actions, and logic live in one
file. The triad pattern (types/actions/reducer) keeps each file focused. Action
creators give consumers a typed API instead of raw `dispatch({type: '...'})`
calls scattered around.

## Steps
1. `types.ts` — `CartState` + `Action = AddItem | RemoveItem | Clear`.
2. `actions.ts` — `addItem(item)`, `removeItem(id)`, `clear()` returning typed objects.
3. `reducer.ts` — pure `cartReducer(state, action)` switching on `action.type`.
4. `CartView.tsx` — `useReducer(cartReducer, initial)` + dispatch calls via action creators.

## Common pitfalls
- Inlining business logic in components — it belongs in the reducer.
- Calling `dispatch({type: ...})` directly — defeats the action creators.
