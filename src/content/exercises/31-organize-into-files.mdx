## Goal
Take a single-file app and split it into `Header.tsx`, `Sidebar.tsx`, `Main.tsx`,
and an `App.tsx` that composes them.

## Concepts
File-per-component is the dominant React convention because it makes navigation
and reviews easier. A short barrel (`components/index.ts`) lets callers write
`import { Header, Main } from "./components"`.

## Steps
1. In `App.tsx`, replace the inline header markup with `<Header />` from `./components/Header`.
2. Do the same for `<Sidebar />` and `<Main />`.
3. Create `components/index.ts` that re-exports all three.
4. Update `App.tsx` to import from the barrel.

## Common pitfalls
- Default-exporting from each file but then importing as a named import — pick one style per file.
- Circular imports — never import the parent from a child.

## Stretch
Add a `Footer` component and slot it after `Main`.
