## Goal
Lay out the project as if it were a tiny monorepo: `app/`, `ui/`, `lib/`,
each importing from each other through path aliases instead of relative paths.

## Concepts
Even single-package projects benefit from the discipline of aliased imports.
You can later promote any aliased folder into a real workspace package without
rewriting any imports.

## Steps
1. `tsconfig.json` paths: `@app/*` → `./app/*`, `@ui/*` → `./ui/*`, `@lib/*` → `./lib/*`.
2. `lib/format.ts` — small currency formatter.
3. `ui/Money.tsx` — uses `@lib/format`.
4. `app/Page.tsx` — uses `@ui/Money`.
5. `App.tsx` renders `<Page />`.

## Common pitfalls
- Mixing `@ui` aliases with `../ui` relative imports — pick one and enforce it.
- Forgetting to update bundler config to mirror tsconfig paths.

## Stretch
Add `lib/api.ts` with a fake fetch and consume it from `@app/Page`.
