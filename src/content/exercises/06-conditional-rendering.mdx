## Goal
Render a fetch-status UI that shows a spinner, error, or the result based on a
discriminated union state.

## Concepts
Discriminated unions let TypeScript prove which branch you're in. If your state
is `{ status: 'loading' }`, the compiler refuses to let you access `data` until
you check `status === 'success'`.

## Steps
1. Define `type Fetch<T> = { status: 'idle' } | { status: 'loading' } | { status: 'error'; error: string } | { status: 'success'; data: T }`.
2. Add buttons to simulate each state.
3. Switch on `state.status` and render the appropriate UI.

## Common pitfalls
- Using optional fields instead of a discriminated union — you lose type safety.

## Stretch
Add a `retry` button that only appears in the `error` state.
