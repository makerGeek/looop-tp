## Goal
Read a complete `route.ts` showing a typed JSON response, Zod-validated search
params, and request-memoized DB reads.

## Concepts
Next 15 flipped fetch defaults: no cache unless you ask. This forces you to be
explicit about what should be cached vs always fresh. `cache()` from React
dedupes reads within a single server request.

## Read-along
1. `app/api/users/route.ts` — exports `GET` and `POST`.
2. Validates `searchParams` with Zod.
3. Uses `cache()` to dedupe `getUsers()` calls within the request.

## Common pitfalls
- Assuming your `fetch` is cached by default — in Next 15 it is not.
- Mixing `cache` (deduping) with `unstable_cache` (full-route cache) — they're different tools.
