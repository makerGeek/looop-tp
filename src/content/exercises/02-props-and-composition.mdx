## Goal
Build a reusable `Card` layout component that accepts a `title` prop and
arbitrary `children`.

## Concepts
Composition beats prop drilling. Rather than a `Card` with a `content: string`
prop, accept `children: React.ReactNode` and let callers project whatever JSX
they want inside.

## Steps
1. Define `interface CardProps { title: string; children: React.ReactNode }`.
2. Render a rounded `<article>` with the `title` as an `<h3>` and the children
   inside a `<div>`.
3. In `App`, use two `<Card>` instances with different inner content.

## Common pitfalls
- Typing children as `string` — it's `React.ReactNode`, which covers elements, arrays, and text.

## Stretch
Add an optional `footer` slot prop (also `React.ReactNode`) and render it below the children.
