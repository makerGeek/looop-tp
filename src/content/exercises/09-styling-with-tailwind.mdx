## Goal
Style a Button that has `primary | secondary | ghost` variants and a hover state.

## Concepts
Tailwind v4 ships a full utility set and uses modern CSS (`@layer`, `:has()`,
custom properties). Use `data-variant` + selectors, or inline utilities, or a
`variants` record. This exercise uses the record pattern because it scales.

## Steps
1. Build a `Button` component that accepts `variant: 'primary' | 'secondary' | 'ghost'`.
2. Map each variant to a utility string.
3. Add a `hover:` color change per variant.
4. Render one of each in `App`.

## Common pitfalls
- Forgetting to purge unused classes — Tailwind v4's Oxide engine handles this automatically in production.

## Stretch
Add a `disabled` state using `disabled:opacity-50 disabled:cursor-not-allowed`.
