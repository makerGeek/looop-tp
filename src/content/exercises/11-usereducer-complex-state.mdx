## Goal
Build a counter with `useReducer` — actions: `increment`, `decrement`, `set`, `reset`.

## Concepts
`useReducer` shines when state is complex or next state depends on multiple
branches. Actions make intent explicit and easy to log/replay.

## Steps
1. Define `Action` as a discriminated union.
2. Write `reducer(state, action)` returning a new state per `action.type`.
3. Wire buttons to `dispatch({ type: '...' })`.

## Common pitfalls
- Mutating state inside the reducer.
- Forgetting the `default` branch — TypeScript's exhaustive check catches this.

## Stretch
Add an `undo` action that restores the previous state (keep a history array).
