## Goal
Rewrite a naive test to use Testing Library queries and assert on user-visible
behavior.

## Concepts
A good test reads like a user: "When I click Add, a todo with 'Buy milk' appears."
Tests that query the DOM by role/label survive refactors; tests that reach into
implementation break every time you change a detail.

## Steps (the sandbox includes a hand-rolled `render` + assertion helpers)
1. Render `<Counter />`.
2. Click the `+` button twice.
3. Assert the heading reads `2`.

## Common pitfalls
- Using `container.querySelector` — breaks on refactors.
- Checking state directly via refs — test behavior, not internals.

## Stretch
Add a test for keyboard interaction: press Enter to increment.
